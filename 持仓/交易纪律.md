# 交易纪律

> 本文件是盘中交易前置检查清单。任何 AI 助手讨论买入、加仓、接回、抄底、做T、减仓或清仓时，先按本文件督促执行，再进入操作建议。

## 一、临时交易冷静30分钟规则

凡是盘中临时产生的买入、加仓、接回、抄底、做T冲动，先不下单。

必须先写清楚四件事：

| 问题 | 必填内容 |
|------|----------|
| 交易属性 | 这笔是短线、中线还是长线？ |
| 买入理由 | 因为什么买？如果理由失效，怎么处理？ |
| 退出规则 | 止损/减仓条件是什么？按收盘还是盘中？ |
| 极端情景 | 如果再跌10%，是按计划加、持有，还是认错走？ |

写完后冷静 **30分钟**。30分钟后仍然想做，再执行。

## 二、短线仓不得自动转中线仓

短线仓的买入理由如果是"未来几天修复""事件后反弹""尾盘V反""做T价差"，那它就是短线仓。

短线仓失败后，禁止用下面这些话自动改成中线仓：

- "反正基本面没坏"
- "最多再跌10%，我能扛"
- "主线还在，等等就回来"
- "跌下来更便宜"

正确处理：

| 情形 | 动作 |
|------|------|
| 短线修复兑现 | 分批兑现，先把短线仓降下来 |
| 未来N个交易日未修复 | 至少减掉短线仓一半 |
| 买入理由被证伪 | 按预设规则认错，不拖成中线 |
| 想改成中线 | 必须重新过一遍中线逻辑和仓位审计，重大仓位先过智囊团 |

## 三、重大决策必过智囊团

以下情况必须先触发 [[智囊团/_章程]]，不过不开仓：

| 情形 | 触发条件 |
|------|----------|
| 新开个股仓位 | 单票新增超过总资产的X%（根据个人风险承受能力设定） |
| 单票集中 | 单票市值超过总资产的X% |
| 改止损线 | 向下放宽止损超过原止损位的X% |
| 清仓集中 | 清掉多只票、满仓一只 |
| 事件博弈 | 赌单一事件（上会、财报、政策发布） |
| 杠杆操作 | 任何形式的融资买入 |
| 账户重大调整 | 季度调仓、账户转型、大幅增减仓位 |

*（用户可根据自身风险偏好将百分比替换为具体数值）*

## 四、止损纪律

### 止损原则

1. **止损一旦设定，除非逻辑发生根本变化，否则不得临时放宽**
2. **到了止损位犹豫=规则被情绪改写**
3. **止损正确的标志不是"卖在最低"而是"保护了账户"**

### 止损处理

| 情形 | 动作 |
|------|------|
| 触及止损位 | 按计划执行，不找理由推迟 |
| 止损后想接回 | 至少等一个交易日，且必须有新的买入信号 |
| 连续止损N次 | 暂停交易，复盘找原因 |

## 五、仓位管理

### 仓位上限

| 类别 | 上限 |
|------|------|
| 单只股票（股票账户） | 不超过总资产的X% |
| 单只ETF（ETF账户） | 不超过ETF账户的X% |
| 同类行业/板块集中 | 合计不超过总资产的X% |
| 总仓位（股票账户+ETF账户合计） | 不超过总资产的X% |
| 现金仓位（下限） | 不低于总资产的X% |

*（用户根据自身风险承受能力填入具体百分比）*

### 加仓原则

1. **亏损仓不因"跌了"而加**——只能因为"证据变强"而加
2. **盈利仓加仓不超过原始仓位**
3. **加仓后统一计算成本和止损位**
4. **仓位上不去时不追高——宁可错过，不可做错**

## 六、买入检查清单

下单前过一遍：

- [ ] 我有明确的买入理由（不是"感觉"或"听别人说"）
- [ ] 当前价格在我的预设买点范围内
- [ ] 我设好了止损位
- [ ] 这笔交易不会让单票仓位超限
- [ ] 我还有足够的现金应对其他机会或风险
- [ ] 如果不是盘中临时冲动（已冷静30分钟），就是计划内交易
- [ ] 产业卡片/个股调研的逻辑当前仍然成立
- [ ] 大盘/板块环境允许操作（非全面派发期）

## 七、卖出/减仓检查清单

- [ ] 止损触发了？——如果是，必须执行
- [ ] 止盈触发了？——如果是，至少减部分
- [ ] 买入理由被证伪了？——如果是，认错
- [ ] 出现更好的机会需要腾仓位？——先清仓再开仓
- [ ] 仓位超过上限需要降风险？——优先减信心最弱的票

## 八、常见错误自查

做决策时对照以下历史错误，避免重复踩坑：

- ❌ **满仓赌单一事件**：没有纠错余地
- ❌ **止损后改规则**：到了止损位犹豫
- ❌ **下跌途中"便宜了"就接**：没有买入信号
- ❌ **追高强趋势股**：产业逻辑对≠买点对
- ❌ **把"排序优"当"买点对"**：好票买贵了照样亏
- ❌ **空仓焦虑推动操作**：资金刚释放就急着买
- ❌ **把讨论当实操**：假设性讨论被误记为真实操作
