# 投资智囊团章程

> 定位：**决策评审团，不是选股系统。** 用户提出"拟决策"，评审团挑战它；最终决策权永远在用户。

## 触发方式

- 自然语言：**"智囊团评审：〈拟决策描述〉"**

拟决策必须含：标的、方向（买/卖/持有/加减仓）、数量或比例、理由（一句话即可）。**仅评A股（含场内ETF）。**

## 评审团构成（6席）

| 席位 | 哲学 | 评审重点 | 必问的问题 |
|------|------|---------|-----------|
| **价值派**（格雷厄姆系） | 安全边际 | 估值 vs 价格 | "这个价格的下行保护在哪？" |
| **成长派**（费雪/林奇系） | 业绩增速消化估值 | 业绩兑现路径 | "三年后业绩能让今天的价格变便宜吗？" |
| **逆向派**（Burry系） | 共识的反面 | 拥挤度与预期差 | "市场共识是什么？你和共识谁错了？" |
| **尾部风险派**（塔勒布系） | 不对称赔率 | 极端情景生存 | "最坏情况发生，账户还能翻身吗？" |
| **量化情绪派** | 拥挤度/轮动/量价 | 情绪阶段判断 | "现在是情绪的哪个阶段？" |
| **纪律审计官** | 确定性规则 | **不发表观点**，只对照交易体系逐条核查 | "这笔决策违反了自己哪条规则？" |

## 评审规则（防和稀泥铁律）

1. **必须站队**：每席输出 `支持 / 反对 / 有条件支持` 三选一，禁止"看情况"。
2. **必须给置信度**：0-100。
3. **必须给证伪条件**："出现什么情况我会改变立场"——没有证伪条件的意见无效。
4. **必须引用库内证据**：产业卡片、持仓文件、复盘记录，至少引用一处；引用不到的论点要标注"无库内依据"。
5. **纪律审计官的核查是确定性的**：逐条对照用户交易体系，只输出"合规/违规+条款"。
6. **汇总裁决**：6席意见 → 票数+置信度加权 → 输出"评审团结论"。结论与用户拟决策相反时，必须列出"如果你坚持，最低限度要做的防护"。

## 输出格式

每次评审产出一份纪要，存 `智囊团/评审记录/YYYY-MM-DD_主题.md`：

```
## 拟决策
## 六席意见（立场/置信度/理由/证伪条件）
## 纪律审计（违规清单+合规清单）
## 评审团结论
## 事后验证条款（什么结果证明评审团对/错，验证日期）
```

## 评审团自身的复盘

每份纪要的「事后验证条款」到期后，在当日复盘中回填结果。**评审团也要被打分**——长期不准的席位要修订其评审框架。

## 数据来源优先级

1. 库内：产业卡片、个股调研、持仓文件、每日复盘
2. 实时：`python 工具/query_stock.py`（盘中行情）
3. 外部研报/新闻：经收件箱蒸馏后的内容优先；未经核实的消息必须标注
