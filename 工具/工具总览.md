# 工具总览

> 本文件是 `工具/` 目录的入口。新 AI 不需要一上来通读所有脚本；先读本文件，再按任务打开对应脚本或说明文档。

## 使用原则

- 工具目录是本库的能力库，不是杂物堆：行情、资金、扫盘、回测、飞书通知都在这里。
- 先判断任务类型，再选工具；不要为了"看起来更全面"乱跑脚本。
- 有副作用的工具要谨慎：飞书推送、拉取外部数据、写日志，执行后按项目铁律落盘并回报。
- 盘中实时数据会联网；涉及把持仓代码发送到外部接口时，遵守当前环境权限和用户授权。

## 一、行情与资金

| 工具 | 何时使用 | 说明 |
|------|------|------|
| `lookup_stock.py` | 核验股票名称/代码 | 查询本地 `stock_names_cache.csv`，只返回命中行，避免整表进入上下文 |
| `query_stock.py` | 查实时价格、涨跌幅、分时 | 腾讯行情接口；适合看价格和分时，不含主力资金；已改为标准库实现，不依赖 requests |
| `fund_flow.py` | 查个股主力资金 | thsdk 同花顺接口；输出现价、涨幅、主力净额、主力净量、5日涨幅、量比、换手 |
| `fetch_fund_flow.py` | 盘后拉钱脉历史数据 | 从小白 API 拉取每日资金流 JSON，落到 `数据/资金流向/` |
| `fund_flow_monitor.py` | 盘中资金监控 | 监控重点股资金变化，可配合飞书推送 |
| `fund_flow_monitor.bat` | Windows 启动封装 | 给计划任务或手动启动使用 |
| `钱脉系统交接文档.md` | 理解钱脉系统 | 新对话遇到"钱脉/资金主线/资金持续性"先读 |
| `小白数据对接规格.md` | 对接小白 API | 修改钱脉采集字段或接口时读 |

## 六、日常使用速查

- 用户提到不确定股票名称/代码：先用 `lookup_stock.py`，多条匹配时不要猜。
- 用户问"现在价格/分时"：用 `query_stock.py`。
- 用户问"资金有没有进/有没有止血"：用 `fund_flow.py`，必要时结合钱脉历史 JSON。
- 用户问"今天能不能买/追/减"：用 `sector_bias_radar.py` 和 `fund_flow.py`。
- 用户问"钱脉/资金主线"：先读 `钱脉系统交接文档.md`。
- 用户问"扫盘/雷达/星级"：先读 `扫盘系统说明.md`。
- 用户问"定时任务/自动任务"：先读 `Windows定时任务管理.md`。
- 用户问"下午盘总结/盘后判断/明天怎么看/持仓给指引"：读 `下午盘复盘输出模板.md`，按模板输出。

## 七、维护提醒

- 新增工具时，同步更新本文件和 `_索引.md`。
- 改资金/扫盘/飞书相关脚本时，先读对应说明文档。
- 工具输出如果形成判断或抓取成果，应按项目规则写入相应日志或复盘文件。
