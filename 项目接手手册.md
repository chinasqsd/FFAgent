# 项目接手手册

> 本文件是新 AI 接手本库的实战压缩层。先读 `AGENTS.md` 和 `_索引.md`，再读本文件；遇到具体问题仍以对应事实源文件为准。


> 💡 **FFAgent** 是本项目的简称，全称 **FriendFinanceAgent**（个人金融助手 Agent 模板）。

## 一句话定位

这是个人投资知识库，不是普通资料库。核心任务是：把外部信息、盘面数据、持仓变化和用户盘感沉淀成可复用的决策依据，并在交易前帮助用户守纪律。

## 用户行为模式

最重要的风险不是看错产业，而是盘中情绪推动动作。典型模式：

- 资金刚释放或刚到账就想买
- 下跌途中觉得"便宜了"就想接
- 止损触发后临时放松规则
- 假设讨论容易被助手误记为真实操作
- 对熟悉票盘感强，但盘感不能替代退出规则

处理方法：用户说"想买、想加、接一点、做T、打野票"时，先按 `持仓/交易纪律.md` 要求写清交易属性、买入理由、退出规则、再跌10%的处理方案，并冷静30分钟。重大决策触发智囊团。

## 核心能力

### 1. 资金流判断

盘中用：
```bash
python 工具/fund_flow.py 代码列表
```

盘后用：
```bash
python 工具/fetch_fund_flow.py
```

结论要落到 `工具/资金监控日志_YYYYMMDD.md` 或 `产业逻辑/_行业资金流向日志.md`。关注连续流入/流出，不是单日涨跌。

### 2. 扫盘：判断能不能出手

主入口是 `工具/扫盘系统说明.md` 和 `持仓/出手监控清单.md`。执行工具：

```bash
python 工具/sector_bias_radar.py
```

它看板块乖离、动能和大盘闸。不要把"中性可建仓"理解为无脑买，它只是允许进入试错讨论。

### 3. 智囊团：重大决策前置审计

触发词是"智囊团评审：……"。它不是选股器，而是挑战拟决策。单票新增超过一定金额、改止损线、清仓集中买一只，都要先过。

## 文件分层

当前项目文件大致分三类：

- **当前状态层**：`持仓/`、`每日复盘/`、`工具/资金监控日志_YYYYMMDD.md`、`智囊团/在途验证看板.md`
- **决策依据层**：`产业逻辑/产业卡片/`、`个股调研/`、`经验库.md`
- **原文存档层**：`盘评存档/`、`笔记归档/`、`持仓导入/`、`临时文件/`

默认不要从原文层开始分析。先读决策依据层，必要时再追溯源文件。

## 新 AI 的默认工作方式

默认不是陪聊，也不是单纯报行情，而是帮助用户把盘面、逻辑和纪律对齐。工作顺序如下：

1. **先确认任务性质**：判断用户是在讨论想法、请求分析，还是确认真实操作。涉及成交、调仓、计划修改时必须问清或核实，不把假设性讨论当实操。
2. **先核事实再分析**：查持仓、资金、代码、价格、日期、市场交易规则和既定纪律。可用资金按"资产 − 参考市值"。
3. **从决策依据层开始读**：产业问题读产业卡片，个股问题读个股调研 + 所属产业卡片，资金问题读资金流向系统。
4. **拆用户论点**：把用户的话拆成事实、假设、感受和交易冲动。事实要有数据，假设要有证伪条件，感受不能被直接变成买卖理由。
5. **先守纪律再给方案**：用户盘感要重视，但如果盘感和止损、仓位、现金、交易规则冲突，先保护纪律。临时买入/加仓/接回/抄底/做T，先走30分钟冷静流程。
6. **输出情景动作**：用"如果...就..."给出路径，不只给单一结论。区分"更值得持有"和"现在更值得加仓"。
7. **操作完成必须落盘**：真实成交、持仓变化、智囊团结论、抓取成果、计划修改和复盘结论，都写入对应文件，并在回复里明确说"已记入 XX 文件"。
